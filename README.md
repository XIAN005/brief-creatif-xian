# Brief créatif de XIAN

Générateur aléatoire de briefs créatifs pour projets de design — branding, logo, affiche, packaging, papeterie, signalétique et plus. Chaque clic génère un « bon de commande » complet : type de projet, client fictif, direction artistique, contrainte créative, livrables, délai, palette de couleurs et suggestion typographique.

## Aperçu

Le générateur se présente sous la forme d'une fiche de commande d'atelier (esthétique carbone / tampon encreur), pensée pour donner un cadre de travail rapide à un designer, un étudiant ou toute personne cherchant un exercice créatif.

## Utilisation

1. Ouvrir `index.html` dans un navigateur (ou visiter l'URL GitHub Pages du repo).
2. Cliquer sur **« Tamponner un nouveau projet »** pour générer un nouveau brief.
3. Cliquer sur **« Copier le brief »** pour copier le texte dans le presse-papiers et le réutiliser ailleurs (notes, outil de conception, etc.).

## Contenu généré

Chaque brief comprend :

- **Type de projet** — identité de marque, logo, affiche, packaging, papeterie, signalétique, identité événementielle, édition
- **Client et secteur** — nom fictif et domaine d'activité
- **Direction artistique** — deux mots-clés d'ambiance (brutaliste, japandi, punk/DIY, luxe discret, etc.)
- **Contrainte créative** — une règle inattendue pour pousser la réflexion
- **Livrables** — ce qui doit concrètement être produit
- **Délai** — du sprint de 2 heures au projet sans deadline
- **Palette** — 4 couleurs nommées avec codes hexadécimaux
- **Typographie** — une suggestion de paire de polices

## Stack technique

Fichier HTML unique, sans dépendance ni build : HTML, CSS et JavaScript vanilla. Polices chargées via Google Fonts (Anton, IBM Plex Mono, Fraunces). Fonctionne entièrement côté client, aucune donnée n'est envoyée à un serveur.

## Hébergement

Ce projet est conçu pour GitHub Pages : le fichier `index.html` à la racine du repo suffit à le mettre en ligne (Settings → Pages → Deploy from a branch → `main` / `root`).
